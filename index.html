Hoochamacallit System

For the Hoochamacallit System project, I took on the ambitious task of developing a simulator to model a network of specialized machines communicating with a backend server. This project, designed as a Thorton Proof-of-Concept, was a deep dive into Inter-Process Communication (IPC) mechanisms within a UNIX/Linux environment. I developed three critical components: the Data Creator (DC), simulating the production machines; the Data Reader (DR), acting as the server monitoring machine statuses; and the Data Corruptor (DX), introduced to test the network's resilience under stress. Written entirely in C, this project challenged me to implement and manage complex IPC mechanisms, such as message queues and shared memory, ensuring robust and efficient communication across the system.
Project can be found here.

Reliable UDP

In my project, I developed a UDP-based file transfer application in C++ for Windows, focusing on reliability and efficiency. A standout feature is the incorporation of a checksum validation mechanism to ensure file integrity, for which I integrated zlib's checksum functionality. This choice underscored my commitment to robust data verification. The application supports dynamic configuration, avoiding hard-coded parameters and includes an automated test to demonstrate its error-handling capabilities. With its ability to accurately measure and display transfer speeds, my project adeptly handles both ASCII and binary file transfers, showcas ing its versatility and precision in real-world scenarios.

Logging Service

For this innovative project, I designed and developed a custom logging service using Python and a test client in C#, intentionally choosing different languages to demonstrate versatility and comply with project requirements. Word Guessing Game

I developed a client-server application which is my Academic Project designed to support multiple users concurrently in a simple yet engaging word guessing game. This application, showcasing my proficiency in C# and .NET framework, was crafted with a WPF client interface allowing users to connect to a server by entering IP address, Port number, a username, and a game time limit. Upon successful connection, the server, a multi-threaded console application also written in C#, dispatches a string of 80 characters and the total number of words to be found within this string, challenging players to identify as many correct words as possible before time expires. Demonstrated on Microsoft Azure to leverage cloud-based scalability and reliability, this project not only tested my technical skills but also emphasized my commitment to user experience and software reliability. Regular commits to GitHub throughout the development process and a structured demonstration in a lab setting highlighted the project's methodical approach and attention to detail.
